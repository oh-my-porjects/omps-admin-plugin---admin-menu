# 菜单元数据 模块事实

## 职责

管理后台菜单与模块后台规格存储公共模块，负责保存、查询、删除模块后台 spec，并聚合左侧菜单。

## 数据表

- `admin_menu_specs`

## API

- `POST /api/admin-menu/specs/{module}`：写入模块后台规格
- `DELETE /api/admin-menu/specs/{module}`：删除模块后台规格
- `GET /api/admin-menu/specs/{module}`：查询模块后台规格
- `GET /api/admin-menu/specs`：列出模块后台规格
- `GET /api/admin-menu/menu`：聚合后台菜单

## 实时事件

用户前端事件：

- 当前模块没有自有用户前端推送事件。

后台提醒事件：

- 当前模块没有自有后台实时提醒事件。

## 测试资产

- `tests/data.yaml` 声明本模块测试涉及的数据表、种子、清理和快照范围。
- 每个 `api-docs.json.apis[].endpoint_key` 都应有对应 `tests/*.test.json`。
- 新增、删除或修改接口时，必须同时更新 `api-docs.json`、测试文件和必要的数据种子。
